=== Release Notes ===

What Works:
- Full single-cycle RV32I datapath implemented in Verilog.
- All 42 user-level RV32I instructions are supported.
- ECALL, EBREAK, PAUSE, FENCE, and FENCE.TSO are treated as halt instructions (PC is frozen).
- Immediate generator handles all instruction formats (I, S, B, U, J).
- ALU supports all required operations (ADD, SUB, AND, OR, XOR, SLL, SRL, SRA, SLT, SLTU).
- Branch unit correctly evaluates all 6 branch conditions using ALU flags.
- Data memory supports byte-addressable loads and stores (LB, LH, LW, LBU, LHU, SB, SH, SW).
- Register file with x0 hardwired to zero.
- Two separate memories used at this stage (instruction memory and data memory) as permitted.
- Basic test cases provided covering the supported instructions.

What Does Not Work / Not Yet Implemented:
- Pipelining (planned for MS3).
- Unified single-ported memory (planned for MS3).
- Hazard detection and forwarding (planned for MS3).
- FPGA testing not performed at this stage (optional for MS2).

Assumptions:
- Instruction memory is initialized from a .hex file generated by RARS.
- Data memory is initialized with a small set of predefined values for testing.
- The halt signal freezes the PC but does not reset the processor.
- Little-endian byte ordering is used throughout.

Known Issues:
- The .hex file path in InstMem.v is currently hardcoded to a local machine path and must be updated before simulation.
